- POSTGRESQL TARAFINDA `USERS`, `BOOKS`, `MEMBERS`, `LOANS`, `LOANREQUESTS` TABLOLARINI OLUŞTURDUM.
- NODE.JS + EXPRESS İLE BACKEND API ALTYAPISINI KURDUM.
- VERİTABANI BAĞLANTISINI TEST ETTİM VE BACKEND İLE POSTGRESQL ARASINDA İLETİŞİM KURDUM.
- İLK CRUD ENDPOİNTLERİNİ TAMAMLADIM:
  - `GET /APİ/BOOKS`
  - `POST /APİ/BOOKS`
- `BOOKS` TABLOSUNA 10 ÖRNEK KİTAP EKLEDİM.
- REACT NATİVE TARAFINDA ANA EKRAN VE BASİT KİTAP LİSTESİ EKRANINI OLUŞTURDUM.
- REACT NAVİGATİON KURULUMUNU YAPIP EKRANLAR ARASI GECİŞİ AKTİF ETTİM.

BU HAFTA YAZDIĞIM KODLARIN NE YAPTIĞI:

- MOBİL UYGULAMA BACKEND'DEN KİTAP LİSTESİNİ ÇEKİP EKRANDA KART YAPISINDA GÖSTERİYOR.
- ANA EKRANDAN KİTAP LİSTESİNE GEÇİŞ, REACT NAVİGATİON STACK ÜZERİNDEN YAPILIYOR.

# Kutuphane Yonetim Sistemi Projesi 2. Hafta Raporu

VERDİĞİM PROJE ÖN RAPORUNDA BELİRTTİĞİM 2. HAFTA MADDELERİNİ TAMAMLADIM.

BU HAFTA ŞU ŞEKİLDE İLERLEDİM:
- BCRYPT KÜTÜPHANESİNİ EKLEDİM, ŞİFRE HASHLEME SİSTEMİ KURDUM.
- JWT KÜTÜPHANESİ KURDUM VE TOKEN TABANLI KİMLİK DOĞRULAMA SİSTEMİ GELİŞTİRDİM.
- POST /API/AUTH/REGISTER ENDPOİNTİNİ OLUŞTURDUM: KULLANICI KAYIT SIRASINDA `USERS` VE `MEMBERS` TABLOLARINI OTOMATIK DOLDURULUYOR.
- POST /API/AUTH/LOGIN ENDPOINTINI OLUŞTURDUM: KULLANICI GİRİŞİNDEN SONRA JWT TOKEN DÖNDÜRÜYOR.
- TOKENIN GEÇERLİLİĞİNİ KONTROL EDEN VERIFYTOKEN MIDDLEWARE YAZDIM (GÜVENLİ BEARER TOKEN AYRIŞTIRILMASI).
- ROL TABANLI ERİŞİM KONTROLÜ (RBAC) SİSTEMİ TAMAMLADIM:
  - ADMİN: TÜM KULLANICI OPERASYONLARINI YÖNETEBİLİR
  - MEMBER: YALNIZ KENDİ HESABINI YÖNETEBİLİR (SELF-ONLY)
- REQUIREADMIN VE REQUIRESELFORADMIN MIDDLEWARE YAZDIM, KORUMALI ENDPOİNTLERE UYGULADIM:
  - GET /API/USERS → SADECE ADMIN
  - POST /API/USERS → SADECE ADMIN
  - GET/PUT/DELETE /API/USERS/:ID → KENDİ HESABI VEYA ADMİN
- USERS VE MEMBERS TABLO AYRIŞTIRMASI UYGULADIM:
  - USERS: SİSTEM KİMLİĞİ (email, şifre, rol)
  - MEMBERS: MEMBER PROFİLİ (ad, telefon, üyelik tarihi) — SADECE MEMBER SELF-İŞLEMLERİ TARAFINDAN DOLDURULUR
  - ADMIN TARAFINDAN OLUŞTURULAN KULLANICILАR MEMBERS TABLOSUNA EKLENMEZ
- .ENV DOSYASINA JWT VE BCRYPT AYARLARINI EKLEDI: JWT_SECRET, JWT_EXPIRES_IN (1 GÜN), BCRYPT_SALT_ROUNDS (10)
- ADMIN HESABI OLUŞTURMA SCRIPT'İ YAZDUM (NPM RUN ADMIN:CREATE)
- MEMBERMODEL.JS YAZDIM: MEMBERS TABLO İŞLEMLERİNİ YÖNETMEK İÇİN

BU HAFTA YAZDIĞIM KODLAR ŞU İŞLEMLERİ YAPIYOR:
- BCRYPT ŞİFRESİ SECURE HASH'E DÖNÜŞTÜRÜLEREK GÜVENLİ BIÇIMDE SAKLANIYOR
- JWT TOKEN'I KULLANICI KİMLİĞİ VE ROL İLE 1 GÜNLÜK SÜRE İÇİN İMZALAYIP DÖNDÜRÜYOR
- VERIFYTOKEN MIDDLEWARE BEARER TOKEN'I DOĞRULAYARAK İSTEĞE KULLANICI BİLGİSİ EKLİYOR
- REQUIREADMIN VE REQUIRESELFORADMIN MIDDLEWARE'İ ROL KONTROL EDEREK YETKISIZ İSTEKLERİ ENGELLİYOR
- MEMBER SELF-KAYIT: USERS VE MEMBERS TABLOLARININ İKİSİNE DE KAYIT EKLEME
- ADMİN OLUŞTUR: SADECE USERS TABLOSUNA KAYIT EKLE (MEMBER AYRIMINI KORUYARAK)
- CREATEADMINUSER.JS SCRIPT'İ KOMUT SATIRINDAN ADMİN HESABI OLUŞTURMAYA IZIN VERİYOR(BAŞLANGIÇ KOMUTUYDU)

BU HAFTA YZ KULLANDIĞIM KISIMLAR:
- THUNDERCLİENT ÜZERİNDEN CRUD İŞLEMLERİNİ TEST EDERKEN BİRTAKIM HATALAR ALDIM BUNLARIN DÜZELTİLMESİ İÇİN TERMİNAL KOMUTU OLARAK YARDIM ALDIM.
- İLK ADMİN KULLANICISINI OLUŞTURMA KISMINDA (CREATEADMİNUSER.JS) YARDIM ALDIM
